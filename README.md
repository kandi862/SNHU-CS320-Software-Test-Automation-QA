# SNHU-CS320-Software-Test-Automation-QA
**Reflection**

Throughout this course, I learned how to make sure my code is both functional and secure by focusing on validation and testing. I used unit testing to check both valid and invalid inputs, which helped catch errors early. Adding validation directly into constructors and methods also made sure that bad data could not be used in the program. Together, these steps helped improve reliability and reduce potential issues.

When it comes to interpreting user needs, I focused on translating requirements directly into code and tests. For example, if a requirement said a field could not be null or had a character limit, I built that into the class and then created tests to confirm it worked. This helped make sure the program matched what the user actually needed instead of just what I assumed.

My approach to designing software became more structured over time. I started by breaking requirements into smaller pieces, then built simple classes to handle each part. From there, I added services to manage the data and created tests to verify everything worked correctly. Keeping things simple and testing each piece along the way made the overall design easier to manage and understand.

Overall, this course helped me see how testing is not just something done at the end, but something that should be part of the entire development process.
